## 二、執行流程

### 2.1 啟動（`agent.main()`）

1. 建立 `~/.agent0` 工作目錄
2. 顯示歡迎資訊、安全區域路徑（`PROGRAM_DIR`）
3. 進入 `while True` 對話迴圈

### 2.2 對話迴圈

```
┌─ 使用者輸入
│
├─ /quit    → 離開
├─ /memory  → 顯示關鍵資訊
├─ /approved→ 顯示已核可路徑
│
└─ 一般輸入 → 進入 AI 處理流程
```

### 2.3 AI 處理流程

```
Step 1: memory.build_context()
        → 組合 key_info + 最近對話歷史

Step 2: gemini.call_gemini(full_prompt, SYSTEM_PROMPT)
        → 呼叫 Gemini API
        → AI 回覆可能含 <tool> JSON 標籤

Step 3: 解析 <tool> 標籤
        → 用 regex 找出所有 <tool>...</tool>
        → JSON 解析出 tool name + input
        → 呼叫 tools.execute_tool()

Step 4: tools.execute_tool()
        → run_command: security.security_check() → subprocess.run()
        → write_file:  security.is_allowed_path() → open().write()

Step 5: 工具結果送回 AI
        → 再次呼叫 Gemini，讓 AI 決定是否需更多工具

Step 6: 輸出最終回覆

Step 7: memory.update_memory() + memory.extract_key_info()
```

## 三、核心模組說明

### 3.1 `src/config.py` — 設定常數

| 變數 | 說明 |
|------|------|
| `WORKSPACE` | `~/.agent0`，外部工作目錄 |
| `GEMINI_API_KEY` | 環境變數讀取 API Key |
| `MODEL` | Gemini 模型名稱 |
| `MAX_TURNS` | 最大對話輪數（影響記憶長度） |
| `PROGRAM_DIR` | `agent0.py` 所在目錄的上一層（因 `config.py` 在 `src/` 內） |

### 3.2 `src/security.py` — 安全審查

#### 路徑提取 — `extract_paths_from_command()`

```
原始指令 → 移除引號內內容 → 用 FILE_ACCESS_PATTERNS 逐一匹配 → 回傳路徑列表
```

**29 條正則模式**，涵蓋：
- 讀取：`cat`, `less`, `grep`, `ls`, `cd`...
- 寫入：`>`, `>>`, `printf`, `echo`...
- 複製/移動/刪除：`cp`, `mv`, `rm`...
- 編輯器：`nano`, `vim`, `vi`
- 路徑模式：`../`, `/abs`, `~/home`

#### 安全判斷 — `is_allowed_path()`

```
路徑 → resolve_path() 轉絕對路徑
     → 是否在 PROGRAM_DIR 內？→ 自動允許
     → 是否在 approved_paths 中？→ 已核可
     → 否則 → 拒絕
```

#### 使用者核可 — `request_approval()`

```
偵測到外部路徑 → 顯示 ⚠️ 安全攔截 → 詢問 y/n/always
  ├─ y       → 本次允許
  ├─ n       → 拒絕
  └─ always  → 記住此路徑（加入 approved_paths）
```

#### 安全審查入口 — `security_check()`

```
extract_paths_from_command() → 過濾出外部路徑
  ├─ 無外部路徑 → 允許
  └─ 有外部路徑 → request_approval() → 依結果允許/拒絕
```

### 3.3 `src/tools.py` — 工具定義與執行

#### run_command

```
security_check() 通過 → subprocess.run(shell=True)
  ├─ stdout + stderr → 回傳 AI
  └─ 例外 → 回傳錯誤訊息
```
注意：Windows 環境下 `shell=True` 使用 `cmd.exe`，不支援 `<<` heredoc。

#### write_file（本版新增）

```
is_allowed_path() 通過 → os.makedirs() → open().write()
  ├─ 成功 → 回傳寫入資訊
  └─ 失敗 → 回傳錯誤訊息
```
優點：繞過 shell，無跨平台相容問題，支援多行內容。

### 3.4 `src/memory.py` — 記憶管理

| 元件 | 說明 |
|------|------|
| `conversation_history` | 近期對話（保留 `MAX_TURNS*4` 則） |
| `key_info` | AI 自動萃取的關鍵資訊 |
| `build_context()` | 組合 memory 成 prompt |
| `update_memory()` | 記錄對話 + 工具結果 |
| `extract_key_info()` | 另一次 Gemini 呼叫，萃取需長期記憶的資訊 |

### 3.5 `src/gemini.py` — Gemini API

```
call_gemini(prompt, system="")
→ _gemini_client.models.generate_content()
→ 回傳 response.text.strip()
```

使用 `google-genai` 套件，單次呼叫無串流。

## 五、資料流示意（以寫入 bubble_sort.c 為例）

```
使用者輸入 "幫我寫一個 bubble_sort.c"
  │
  ▼
memory.build_context()
  │  ┌─ key_info: []
  │  └─ conversation_history: [...]
  ▼
gemini.call_gemini("User: 幫我寫一個 bubble_sort.c", SYSTEM_PROMPT)
  │
  ▼
AI 回覆：
  <tool>
  {"name": "write_file", "input": {"path": "bubble_sort.c", "content": "#include..."}}
  </tool>
  │
  ▼
tools.execute_tool("write_file", {"path": "bubble_sort.c", "content": "..."})
  │
  ├─ security.is_allowed_path("bubble_sort.c")
  │    → 在 PROGRAM_DIR 內 → 自動允許
  │
  ├─ open("bubble_sort.c", "w").write("...")
  │
  └─ 回傳 "✅ 已寫入 ..."
  │
  ▼
AI 最終回覆："已建立 bubble_sort.c"
```

## 六、安全區域判定

```
PROGRAM_DIR = /path/to/_ml/05/
                    │
                    ├── agent0.py       → 安全（在 PROGRAM_DIR 內）
                    ├── src/            → 安全（子目錄）
                    ├── _doc/           → 安全（子目錄）
                    ├── bubble_sort.c   → 安全（在 PROGRAM_DIR 內）

                    ├── C:/Windows/     → 不安全 ❌
                    ├── /etc/passwd     → 不安全 ❌
                    └── ~/.agent0/      → 不安全 ❌（需核可）
```

## 七、跨平台注意事項

| 項目 | Linux/macOS | Windows |
|------|-------------|---------|
| shell | `/bin/sh`（支援 `<<`） | `cmd.exe`（不支援 `<<`） |
| 寫檔建議 | `write_file` 工具或 `cat << EOF` | **僅用 `write_file` 工具** |
| Python 指令 | `python3` | `python` 或 `py` |
